# ipyrad2 Documentation

> Draft: This is the first scaffold for the ipyrad2 documentation site. Use it to organize writing, not as a finished reference.

## Summary

This site will document the ipyrad2 command-line workflow for RAD-seq assembly, output export, and downstream analysis.

## Who This Documentation Is For

- New users trying to understand the workflow from raw reads to analysis.
- Existing ipyrad users comparing the original project to ipyrad2.
- Contributors who need a stable map of commands, inputs, outputs, and design choices.

## Major Sections

- Getting Started
- Assembly
- Writing Outputs
- Recipes
- Analysis Guide

## Recommended Reading Order

1. Start with [What Is ipyrad?](./getting-started/what-is-ipyrad.md), then [The ipyrad Ethos](./getting-started/ethos.md).
2. Read [Installation](./getting-started/installation.md) and [Files and Data Types](./getting-started/files-and-data-types.md).
3. Read [Quick Guide](./assembly/index.md) for the end-to-end workflow.
4. Read the command pages that match the step you are running.
5. Use Writing Outputs and [Analysis Guide](./analyses/index.md) once you reach filtered SNP or sequence data.
6. Use [Recipes](./recipes/index.md) once that section is filled in for example-driven workflows that connect export choices to specific downstream tools.
